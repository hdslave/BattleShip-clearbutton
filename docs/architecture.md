# Architecture

## Source Organization
```
decomp/                — Submodule: JRickey/ssb-decomp-re#port-patches
  include/             — C decomp shim headers (PR/, sys/, n_audio/, libc/, ...)
  src/
    sys/               — System: main loop, DMA, scheduling, audio, controllers, threading
    ft/                — Fighters: per-character logic (ftmario/, ftkirby/, ftfox/, etc.)
    sc/                — Scene management
    gm/                — Game modes
    gr/                — Graphics/stage rendering
    mn/                — Menus
    it/                — Items
    ef/                — Effects/particles
    lb/                — Library utilities
    mp/                — Maps
    wp/                — Weapons/projectiles
    mv/                — Movie/cutscene
    if/                — Interface/HUD
    credits/           — Credits sequence
    libultra/{gu,n_audio,audio}/  — Decomp N64 SDK files the port still compiles
    relocData/         — Decomposed reloc data (compile-from-source target, M2/M3)
  tools/relocFileDescriptions.us.txt, relocData.py, fixRelocChain.py
```

## Port Target Structure
```
port/             — Modern C++ port layer (Ship::Context, resource factories, bridges)
include/reloc_data.h — Sole port-tree header (auto-generated by tools/generate_reloc_stubs.py)
tools/            — Port-side codegen + helper scripts
yamls/us/         — Torch YAML asset extraction configs
libultraship/     — Git submodule
torch/            — Git submodule
decomp/           — Git submodule (decomp source on port-patches branch)
CMakeLists.txt
```

## Key Dependencies
- **libultraship** (Kenix3/libultraship) — replaces libultra with PC-native rendering, audio, input, resource management
- **Torch** (HarbourMasters/Torch) — asset extraction from ROM into .o2r archives via YAML configs
- **Fast3D** — translates N64 F3DEX2 GBI display lists to OpenGL/D3D11/D3D12/Metal
- **SDL2** — windowing, input, audio backend
- Reference ports: HarbourMasters/Starship (Star Fox 64), HarbourMasters/SpaghettiKart (Mario Kart 64)

## What NOT to Include in Port
- `decomp/src/ovl8/`, `decomp/src/db/` — Debug overlay/menu (kept on the
  `port-patches` branch of the decomp submodule only as headers transitively
  required by `sys/develop.h`; no `.c` files compiled)
- `decomp/src/audio/`, `decomp/src/particles/` — Asset authoring sources;
  the port consumes these via Torch instead. Removed from `port-patches`.
- `decomp/src/libultra/{io,libc,os,sp,vimodes}/` — Replaced by libultraship.
  Removed from `port-patches`. Port still compiles `libultra/{gu,n_audio,audio}/`.
- Assembly files (`asm/`) — Port uses C code only
- N64 ROM or copyrighted assets — Users provide their own ROM
